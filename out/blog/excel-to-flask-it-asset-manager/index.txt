<h1>From Excel to Enterprise Web App: How I Built an IT Asset Manager in Python</h1>
<p>"Can you tell me how many unassigned laptops we have?" Simple question. Before I built this app, answering it required opening a shared Excel file, scrolling through hundreds of rows, applying filters, hoping the data hadn't been corrupted by whoever last edited it, and spending 10 minutes to surface a number that should take 10 seconds.</p>
<p>That question — asked by my IT manager on a Monday morning — is why I built the IT Asset Manager.</p>

<h2>The Excel Problem</h2>
<p>The shared Excel file had been the system of record for hardware assets since anyone could remember. On paper, it worked. In practice:</p>
<ul>
<li><strong>No audit trail.</strong> If a serial number changed, no one knew when or why.</li>
<li><strong>Duplicate serial numbers.</strong> Two people adding the same device at the same time created corrupted records.</li>
<li><strong>No validation.</strong> Every field was free text — typos, inconsistent formatting, missing mandatory data.</li>
<li><strong>Terrible search.</strong> Ctrl+F in a 580-row spreadsheet across 12 columns is not a workflow.</li>
<li><strong>Broken exports.</strong> Sending the Excel file to management meant formatting it manually every time.</li>
<li><strong>No dashboard.</strong> Low stock? End-of-life devices? Warranty expiry? All invisible.</li>
</ul>
<p>The Excel file wasn't a system — it was a shared text document pretending to be a database.</p>
<hr>
<h2>Building the Replacement</h2>
<p>I chose Python + Flask for the backend because I needed something I could build and deploy quickly on my local Mac without any server infrastructure. SQLite for the database — appropriate for a single-location asset register with ~1,500 records and no concurrent write conflicts.</p>
<h3>The Data Model</h3>
<p>The core <code>assets</code> table captures everything relevant to an enterprise IT asset:</p>
<pre><code class="language-sql">CREATE TABLE assets (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    asset_type    TEXT NOT NULL,        -- Laptop, iPhone, iPad, Monitor, Printer
    status        TEXT NOT NULL,        -- Assigned, Unassigned, In Repair, Retired
    serial_no     TEXT UNIQUE NOT NULL, -- Enforced uniqueness at DB level
    sap_eq        TEXT UNIQUE,          -- SAP equipment number (ERP reference)
    model         TEXT NOT NULL,
    assigned_to   TEXT,
    location      TEXT,
    purchase_date TEXT,
    warranty_exp  TEXT,
    notes         TEXT,
    created_at    TEXT,
    updated_at    TEXT
);

CREATE TABLE asset_history (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    asset_id   INTEGER,
    field      TEXT,
    old_value  TEXT,
    new_value  TEXT,
    changed_at TEXT
);
</code></pre>
<p>The <code>UNIQUE</code> constraint on <code>serial_no</code> does what Excel never could — it makes duplicate serial numbers a database error, not a user error.</p>
<h3>Smart Excel Import</h3>
<p>The most-used feature after the dashboard. The import logic handles two modes:</p>
<p><strong>Add Only</strong> — safe for regular use. New records from the Excel file are inserted; existing serial numbers are skipped. Zero risk of overwriting live data.</p>
<p><strong>Add + Update</strong> — for bulk refreshes. New records are inserted; existing records are updated if any field has changed. A history entry is written for every change.</p>
<p>Both modes filter garbage headers — Excel files from vendors often have metadata rows at the top. The import skips any row where the first cell doesn't look like an asset type.</p>
<pre><code class="language-python">def smart_import(file, mode="add_only"):
    wb = load_workbook(file)
    ws = wb.active
    imported, skipped, updated = 0, 0, 0

    for row in ws.iter_rows(min_row=2, values_only=True):
        if not is_valid_asset_row(row):
            skipped += 1
            continue
        
        existing = db.execute(
            "SELECT * FROM assets WHERE serial_no = ?", [row[2]]
        ).fetchone()

        if existing and mode == "add_only":
            skipped += 1
        elif existing and mode == "add_update":
            update_asset_with_history(existing, row)
            updated += 1
        else:
            insert_asset(row)
            imported += 1

    return imported, skipped, updated
</code></pre>
